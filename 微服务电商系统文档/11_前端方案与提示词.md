# 11 前端方案与提示词（够用即可，别过度投入）

> **先说定位**：这个项目的简历价值在后端。前端的唯一任务是「把流程演示出来」——
> 能登录、能看商品、能下单付款、能秒杀，录屏/演示时看得见就行。
> **不要**追求 UI 精美、不要花超过总时间的 15%。Element Plus 默认样式直接用，丑一点没关系。

---

## 1. 技术栈（现代、面试友好、够简单）

| 用途 | 选型 | 一句话说明 |
|---|---|---|
| 框架 | Vue 3 + Vite | 主流前端框架；Vite 是它的脚手架/启动工具，启动飞快 |
| 路由 | Vue Router | 管"点哪个链接跳哪个页面" |
| 状态管理 | Pinia | 存全局共享数据，比如"当前登录用户、令牌" |
| UI 组件库 | Element Plus | 现成的按钮/表格/弹窗，不用自己画 |
| 请求 | Axios | 前端向后端发请求的工具 |

> 只做**一个** PC 网页商城即可，覆盖买家主流程；管理后台是可选项，时间紧就不做、用接口文档(Knife4j)代替演示。
> 不要同时做"商城 + 后台 + 小程序"三个端,那是给自己挖坑。

## 2. 前端术语速查（不认识就回这看）

| 词 | 一句话解释 |
|---|---|
| SPA / 单页应用 | 整个网站就一个页面，切换内容靠 JS 局部刷新，不整页跳转 |
| 组件 | 可复用的 UI 积木（一个按钮、一个商品卡片都是组件） |
| 路由 / Router | URL 和页面的对应关系（`/login` 显示登录页） |
| Pinia / 状态管理 | 一个全局"小仓库"，存多个页面都要用的数据（如登录令牌） |
| Axios / 请求 | 向后端要数据、提交数据的工具 |
| 拦截器 (interceptor) | 在每个请求发出前 / 响应回来后统一做点事（如自动带令牌、统一报错） |
| 跨域 / CORS | 浏览器的安全限制：前端和后端域名/端口不同就会被拦，需要后端放行（我们在网关已处理） |
| npm | 装前端依赖包的工具（类似 Java 的 Maven） |
| .env | 存配置的文件（比如后端地址），换环境改这里不用改代码 |
| 响应式 | 数据一变，页面自动跟着更新（Vue 的核心特性） |

## 3. 前端怎么和后端对接（3 个关键点，务必懂）

1. **所有请求都发给网关**：前端 Axios 的基础地址(baseURL)设成网关地址 `http://localhost:9000`，
   不要直接连各个服务。网关再转发给对应服务。
2. **令牌(JWT)的处理流程**：登录成功 → 后端返回令牌 → 前端存进 Pinia + 浏览器 →
   之后每个请求由 **Axios 请求拦截器**自动在请求头加上 `Authorization: Bearer 令牌` →
   如果某次返回"未登录/令牌失效"，**响应拦截器**自动跳回登录页。
3. **统一处理后端的 Result**：后端都返回 `{code, msg, data}`。用 **Axios 响应拦截器**统一拆：
   code=200 就把 data 交给页面；否则用 Element Plus 弹一个错误提示，页面代码就不用每次都判断了。

## 4. 页面清单（每个页面对应哪个后端流程）

| 页面 | 对应后端 | 演示什么 |
|---|---|---|
| 登录 / 注册 | user-service | 拿到令牌、跳首页 |
| 商品列表（首页） | product-service | 分类、分页浏览 |
| 商品详情 | product-service | 看 SKU、加入购物车 |
| 购物车 | cart-service | 改数量、删除、去结算 |
| 确认订单 / 下单 | order-service | 选地址、提交、生成订单号 |
| 收银台（模拟支付） | payment-service | 点"支付"→ 轮询/刷新看订单变"已支付" |
| 我的订单 | order-service | 订单列表、状态、取消 |
| 秒杀页 | seckill-service | 倒计时、点抢 → 拿 token → 轮询结果 |
| 收货地址管理 | user-service | 地址增删改 |
| （可选）管理后台 | product / seckill | 商品上下架、配秒杀场次 |

> **秒杀页要特别注意**：点"抢"之后后端是异步处理的，会先返回一个 token，
> 前端要拿这个 token **每隔 1 秒去查一次结果接口**，查到"成功"显示订单号、"售罄"提示抢光了。
> 这个"提交后轮询结果"的交互，正好对应后端的削峰设计，演示时讲出来很加分。

## 5. 项目结构

```
beibu-mall-web/
├── .env                  # 后端网关地址等配置
├── src/
│   ├── api/              # 按模块封装接口：user.js / product.js / order.js ...
│   ├── components/       # 复用组件：商品卡片、导航栏...
│   ├── views/            # 页面：Login.vue / ProductList.vue / Cart.vue ...
│   ├── router/           # 路由配置
│   ├── store/            # Pinia：用户/令牌
│   ├── utils/request.js  # Axios 封装 + 两个拦截器（最重要的文件）
│   ├── App.vue
│   └── main.js
```

## 6. 前端开发节奏（穿插进 12 周，别单独占用大块时间）

- **第 2~3 周**（后端有了用户+商品后）：搭前端骨架 + 登录页 + 商品列表。**目的是早点看到东西在动，给自己正反馈。**
- **第 5~7 周**：随后端进度补 购物车 → 下单 → 支付 → 我的订单。
- **第 9 周**：做秒杀页（含轮询）。
- **收尾**：简单美化 + 录一段完整流程演示视频（放简历/GitHub）。

> 管理后台、地址管理这些"锦上添花"的，全程最后有空再说，没空就砍。

## 7. 前端全套提示词（复制即用）

> 和后端一样：**开新对话先发【前端开场设定】**，再发具体任务。

### 第 0 步：前端开场设定（每次新对话先发）

```
你是一名资深前端工程师，带我做一个微服务电商项目「北部湾海鲜商城」的前端。
我做过一点后端，但前端基础很弱，很多术语不懂。这个前端只要"够用、能演示完整购物流程"即可，
不追求精美，请帮我用最简单直接的方式实现。

技术栈（务必用这些）：Vue 3 + Vite + Vue Router + Pinia + Element Plus + Axios。

约定：
- 所有请求发给后端网关，地址 http://localhost:9000，写在 .env 里，不要硬编码；
- 后端接口统一返回 {code, msg, data}，code=200 是成功；
- 登录后后端返回 JWT 令牌，之后每个请求要在请求头带 Authorization: Bearer 令牌。

教学要求：每给一段代码，先用大白话讲它在做什么、为什么这么写，遇到术语顺手解释一句；
一次只做一块，不要一次性把整个前端都生成出来。

明白请回复"明白，我们开始吧"，然后等我发任务。
```

### 提示词 F1：搭骨架 + 请求封装（最关键）

```
按我们的前端设定，帮我创建项目骨架，并重点实现 src/utils/request.js（Axios 封装）。要求：
1. 用 Vite 创建 Vue3 项目的命令、要装哪些依赖、怎么启动，命令写全能直接复制；
2. 配好 Vue Router 和 Pinia 的基础结构；
3. 在 request.js 里封装 Axios：
   - 基础地址从 .env 读取（默认 http://localhost:9000）；
   - 请求拦截器：自动从 Pinia/本地取令牌，加到请求头；
   - 响应拦截器：code=200 就直接返回 data；否则用 Element Plus 弹错误提示；
     如果是"未登录/令牌失效"，自动跳转到登录页。
请先讲清"拦截器"是什么、为什么这么封装能让后面写页面省事，再给代码。
```
