# 北部湾海鲜微服务商城（BeibuMall）— 项目文档总览

> 一个用于**校招/秋招简历**的企业级微服务电商系统。卖点是「北部湾活鲜/冰鲜/冻品/干货」，
> 用海鲜的领域特性（活鲜按重计价、时令预售、库存超卖、冷链时效）自然引出微服务里
> 最被面试官追问的硬核技术点，避免"为了微服务而微服务"。

## 1. 这个项目要证明你会什么（简历定位）

招聘方看一个学生的"微服务项目"，本质是想确认三件事，本项目逐一对应：

| 招聘方关心的 | 本项目用什么证明 |
|---|---|
| 你懂**服务拆分**，不是把单体改个名 | 按 DDD 限界上下文拆 8 个服务，每服务独立库（详见 02、03） |
| 你能处理**分布式难题** | 分布式事务(Seata)、分布式锁(Redisson)、库存超卖、缓存三剑客、限流熔断(Sentinel)、秒杀(详见 05) |
| 你有**工程素养** | 企业级测试金字塔、JaCoCo 覆盖率门禁、CI/CD、K8s 部署、可观测性(详见 06、07) |

**一句话简历描述（可直接用）：**
> 基于 Spring Cloud Alibaba 设计并实现的海鲜电商微服务系统，包含用户、商品、库存、订单、
> 支付、购物车、搜索、秒杀 8 个服务；通过 Seata 解决下单扣库存的分布式事务，Redisson +
> Lua 防止秒杀超卖，Sentinel 实现限流降级；建立单元/集成/契约/性能四层测试体系，行覆盖率
> ≥70%，并用 Docker + K8s + Prometheus/Grafana/SkyWalking 完成容器化部署与全链路可观测。

> ⚠️ 诚信提醒：简历只写你**真正做过且能讲清楚**的部分。这套文档帮你把它们做出来、讲明白。

## 5. 面试讲述脚本（STAR，背熟）

**问：你这个项目最难的点是什么？**
> 下单这个动作要同时扣库存、生成订单、扣减优惠券，跨了三个服务三个库。一开始我用本地事务，
> 结果库存扣了订单没生成就会超卖。我引入 Seata AT 模式做全局事务，把这三步绑成一个全局事务，
> TC 协调提交/回滚。但秒杀场景下 Seata 的全局锁会拖垮吞吐，所以秒杀链路我改成"Redis Lua
> 预扣 + RocketMQ 异步落库"，最终一致性，压测下单 TPS 从约 200 提到约 2000。

**问：库存超卖你怎么防的？**
> 普通商品走 DB 乐观锁（`stock = stock - n WHERE stock >= n`）+ Seata；秒杀走 Redis +
> Lua 脚本原子判断并扣减，扣成功再发 MQ 异步创建订单，DB 层再加唯一索引兜底防重复下单。

更多脚本见 `05` 文末「面试问答」与 `06` 文末「测试相关问答」。
