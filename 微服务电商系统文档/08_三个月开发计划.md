# 08 三个月开发计划（12 周）

> 原则：**先打通一条最小主链路（登录→看商品→下单→支付），再逐个加深难点**。
> 不要一开始就追求 8 个服务齐全——先 2~3 个服务跑通端到端，建立正反馈，再扩。
> 每周末产出一个"可演示 + 有测试 + 有 commit"的增量。

## 阶段一：地基（第 1–3 周）

**第 1 周｜环境与骨架**
- 装 JDK21、Docker、IDEA；Compose 起 Nacos/MySQL/Redis。
- 建 Maven 多模块骨架（父 POM + mall-common + gateway + user）。
- mall-common：Result、BizException、全局异常处理、统一日志。
- 交付：项目能起来，Nacos 能看到注册，`/health` 通。

**第 2 周｜用户服务 + 网关**
- user-service：注册/登录/JWT/地址 CRUD；Knife4j 接入。
- gateway：路由 + JWT 鉴权 + 白名单 + 跨域。
- 测试：user 的 service 单元测试（注册重复、密码错误等分支）。
- 交付：能注册登录、带 token 调通过网关。

**第 3 周｜商品服务**
- product-service：分类树、SPU/SKU、详情、分页、上下架。
- 引入 MyBatis-Plus、逻辑删除、自动填充。
- 测试：单元测试 + 第一个 Testcontainers 集成测试(连真 MySQL)。
- 交付：能浏览商品列表与详情。

## 阶段二：核心交易链路（第 4–7 周）

**第 4 周｜库存服务 + 乐观锁防超卖**
- inventory-service：库存查询、乐观锁扣减、预占/释放/实扣、流水。
- 测试：⭐并发扣减集成测试（多线程，断言不超卖）。
- 交付：库存能正确扣减且并发安全。

**第 5 周｜购物车 + 下单（先不上 Seata）**
- cart-service（Redis Hash）。
- order-service：下单(Feign 调 product 算价、调 inventory 预占)、订单详情/列表。
- 交付：加购→下单→生成订单→库存预占，主链路打通（暂用本地事务/手动补偿占位）。

**第 6 周｜Seata 分布式事务**
- 接入 Seata Server + 各库 undo_log；下单方法 `@GlobalTransactional`。
- 验证：故意让某分支失败，确认全局回滚、库存不被错扣。
- 交付：跨服务下单强一致。

**第 7 周｜支付 + 订单状态机 + 超时取消**
- payment-service：发起支付(支付宝沙箱或mock)、回调验签+幂等、发 MQ。
- order：消费"支付成功"转状态；RocketMQ 延时消息做超时取消+释放库存。
- 交付：完整 下单→支付→已支付 / 超时→自动取消 闭环。

## 阶段三：难点与亮点（第 8–10 周）

**第 8 周｜Redis 缓存 + 三剑客 + 分布式锁**
- 商品/SKU 缓存；处理穿透(空值/布隆)、击穿(互斥重建)、雪崩(随机TTL)。
- Redisson 分布式锁（领券/定时任务）。
- 测试：缓存命中/失效路径单测。

**第 9 周｜秒杀服务 ⭐**
- 库存预热、Lua 原子扣减、限流(Sentinel)、去重、MQ 异步落库、结果轮询。
- 测试：⭐Lua 扣减集成测试 + JMeter 压测（优化前后对比数据）。
- 交付：秒杀不超卖、有压测数据。

**第 10 周｜搜索 + 限流降级完善**
- search-service：商品同步 ES（MQ 增量）、检索/筛选/高亮。
- Sentinel：核心接口限流、Feign 熔断降级兜底完善。
- 交付：搜索可用、主链路有容错。

## 阶段四：工程化与收尾（第 11–12 周）

**第 11 周｜测试补齐 + 覆盖率门禁 + CI**
- 补单元/集成测试到行覆盖率 ≥70%（核心 ≥80%）。
- 契约测试(order↔inventory)；JaCoCo 门禁；GitHub Actions 跑通。
- 交付：CI 绿灯、覆盖率报告。

**第 12 周｜容器化/K8s + 可观测 + 文档简历**
- Dockerfile + Compose 一键起；核心服务上 k3s/minikube + Ingress。
- Prometheus/Grafana/SkyWalking 接入并截图。
- 写项目 README（架构图、难点、压测数据、覆盖率、监控截图）；打磨简历描述与面试脚本。
- 交付：可演示的完整项目 + 一份能讲的简历。

## 里程碑检查点（自检）

| 时间 | 必须达成 |
|---|---|
| 第 3 周末 | 能登录、看商品（2 服务 + 网关跑通） |
| 第 7 周末 | 完整交易闭环（下单→支付→状态流转，含 Seata） |
| 第 9 周末 | 秒杀有压测数据、不超卖 |
| 第 12 周末 | CI 绿灯 + 覆盖率达标 + 容器化 + 监控截图 + 简历定稿 |

## 时间不够时的优先级（砍单优先级，从下往上砍）

保命主线（绝不能砍）：用户/商品/库存/订单/支付 + Seata + 秒杀 + 测试体系。
可砍/简化：搜索(可只做 DB 模糊查代替 ES)、契约测试、K8s(退回 Compose)、混沌测试、前端(只做能演示)。
