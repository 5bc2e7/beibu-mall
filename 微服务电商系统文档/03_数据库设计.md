# 03 数据库设计

> 原则：**每服务独立库**，库名见 02。表用 InnoDB、utf8mb4。统一含 `id BIGINT 主键`、
> `create_time`、`update_time`、`deleted TINYINT`(逻辑删除) 字段（下文为节省篇幅多数省略这三个公共字段，
> 实际建表都要带，MyBatis-Plus 用 `@TableLogic` 和自动填充处理）。

## 1. mall_user（用户库）

```sql
CREATE TABLE t_user (
  id           BIGINT PRIMARY KEY AUTO_INCREMENT,
  username     VARCHAR(50)  NOT NULL,
  password     VARCHAR(100) NOT NULL,         -- BCrypt 哈希，不存明文
  phone        VARCHAR(20),
  nickname     VARCHAR(50),
  member_level TINYINT DEFAULT 0,             -- 0普通 1银 2金
  status       TINYINT DEFAULT 1,             -- 1正常 0禁用
  UNIQUE KEY uk_username(username),
  UNIQUE KEY uk_phone(phone)
);

CREATE TABLE t_address (
  id          BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id     BIGINT NOT NULL,
  receiver    VARCHAR(50),
  phone       VARCHAR(20),
  province    VARCHAR(50), city VARCHAR(50), district VARCHAR(50),
  detail      VARCHAR(200),
  is_default  TINYINT DEFAULT 0,
  KEY idx_user(user_id)
);
```

## 2. mall_product（商品库）

```sql
CREATE TABLE t_category (
  id        BIGINT PRIMARY KEY AUTO_INCREMENT,
  parent_id BIGINT DEFAULT 0,
  name      VARCHAR(50),
  level     TINYINT,
  sort      INT DEFAULT 0
);

CREATE TABLE t_spu (
  id           BIGINT PRIMARY KEY AUTO_INCREMENT,
  category_id  BIGINT NOT NULL,
  name         VARCHAR(100),                  -- 北部湾大对虾
  origin       VARCHAR(50),                   -- 产地：广西北海
  is_fresh     TINYINT DEFAULT 0,             -- 1活鲜
  price_type   TINYINT DEFAULT 0,             -- 0按件 1按重(g)
  description  TEXT,
  status       TINYINT DEFAULT 0,             -- 0下架 1上架
  KEY idx_cat(category_id)
);

CREATE TABLE t_sku (
  id          BIGINT PRIMARY KEY AUTO_INCREMENT,
  spu_id      BIGINT NOT NULL,
  spec        VARCHAR(100),                   -- 鲜活/500g
  price       DECIMAL(10,2),                  -- 按件为单价；按重为每单位(如每500g)价
  unit        VARCHAR(10),                    -- 件 / g
  shelf_life  INT,                            -- 保质期(小时)，活鲜很短
  min_buy     INT DEFAULT 1,                  -- 起售量
  img         VARCHAR(255),
  KEY idx_spu(spu_id)
);
```

## 3. mall_inventory（库存库）

```sql
CREATE TABLE t_inventory (
  id          BIGINT PRIMARY KEY AUTO_INCREMENT,
  sku_id      BIGINT NOT NULL,
  total       INT NOT NULL DEFAULT 0,         -- 总库存
  available   INT NOT NULL DEFAULT 0,         -- 可用库存
  locked      INT NOT NULL DEFAULT 0,         -- 预占库存
  version     INT NOT NULL DEFAULT 0,         -- 乐观锁版本号
  UNIQUE KEY uk_sku(sku_id)
);

-- 库存流水，用于对账与排查超卖
CREATE TABLE t_inventory_log (
  id        BIGINT PRIMARY KEY AUTO_INCREMENT,
  sku_id    BIGINT NOT NULL,
  order_no  VARCHAR(64),
  type      TINYINT,                          -- 1预占 2实扣 3释放 4回补
  quantity  INT,
  KEY idx_sku(sku_id), KEY idx_order(order_no)
);
```

**乐观锁扣减（普通商品）核心 SQL：**
```sql
UPDATE t_inventory
SET locked = locked + #{qty}, available = available - #{qty}, version = version + 1
WHERE sku_id = #{skuId} AND available >= #{qty} AND version = #{version};
-- 返回影响行数=0 表示库存不足或并发冲突，业务层判定失败
```

## 4. mall_order（订单库）

```sql
CREATE TABLE t_order (
  id            BIGINT PRIMARY KEY AUTO_INCREMENT,
  order_no      VARCHAR(64) NOT NULL,         -- 业务订单号(雪花/分布式ID)
  user_id       BIGINT NOT NULL,
  total_amount  DECIMAL(10,2),
  pay_amount    DECIMAL(10,2),                -- 实付(减券后)
  status        TINYINT,                      -- 见状态机
  address_snapshot VARCHAR(500),              -- 下单时地址快照(JSON)
  pay_time      DATETIME, cancel_time DATETIME,
  UNIQUE KEY uk_order_no(order_no),
  KEY idx_user(user_id)
);

CREATE TABLE t_order_item (
  id        BIGINT PRIMARY KEY AUTO_INCREMENT,
  order_no  VARCHAR(64) NOT NULL,
  sku_id    BIGINT, spu_name VARCHAR(100), spec VARCHAR(100),
  price     DECIMAL(10,2), quantity INT, img VARCHAR(255),
  KEY idx_order(order_no)
);
```
**订单状态机**：`0待支付 → 1已支付 → 2待发货 → 3已发货 → 4已完成`，分支 `5已取消 6退款中 7已退款`。
状态流转用枚举 + 校验，禁止非法跳转。

## 5. mall_payment（支付库）

```sql
CREATE TABLE t_payment (
  id          BIGINT PRIMARY KEY AUTO_INCREMENT,
  order_no    VARCHAR(64) NOT NULL,
  pay_no      VARCHAR(64),                    -- 第三方流水号
  amount      DECIMAL(10,2),
  channel     TINYINT,                        -- 1支付宝沙箱 2mock
  status      TINYINT,                        -- 0待支付 1成功 2失败
  notify_data TEXT,                           -- 回调原文，便于对账排查
  UNIQUE KEY uk_order_no(order_no)            -- 一单一支付，保证幂等
);
```

## 6. mall_seckill（秒杀库）

```sql
CREATE TABLE t_seckill_activity (
  id         BIGINT PRIMARY KEY AUTO_INCREMENT,
  sku_id     BIGINT NOT NULL,
  seckill_price DECIMAL(10,2),
  stock      INT,                             -- 秒杀库存(独立于普通库存)
  start_time DATETIME, end_time DATETIME,
  status     TINYINT                          -- 0未开始 1进行中 2已结束
);

CREATE TABLE t_seckill_order (
  id          BIGINT PRIMARY KEY AUTO_INCREMENT,
  activity_id BIGINT, user_id BIGINT, order_no VARCHAR(64),
  UNIQUE KEY uk_user_activity(user_id, activity_id) -- 防一人多抢
);
```

## 7. Redis Key 规范

| 用途 | Key | 结构 | 说明 |
|---|---|---|---|
| 购物车 | `cart:{userId}` | Hash | field=skuId, value=数量 |
| 商品缓存 | `product:sku:{skuId}` | String(JSON) | 设过期+随机抖动防雪崩 |
| 秒杀库存 | `seckill:stock:{activityId}` | String(数值) | Lua 预扣 |
| 秒杀去重 | `seckill:bought:{activityId}` | Set | 已购用户 |
| 分布式锁 | `lock:order:{orderNo}` | Redisson | RLock |
| 接口幂等token | `idem:{token}` | String | 一次性 |

## 8. 分布式 ID

订单号/支付号用**雪花算法**（如 Hutool `IdUtil` 或美团 Leaf 思路），不要用数据库自增暴露量，
也便于分库分表演进。可在面试讲"为什么不用自增 ID 当业务单号"。
